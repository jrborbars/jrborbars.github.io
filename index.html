WatchMe - Relógio que mede batimentos cardíacos

Esse projeto se trata de um relógio que mostra no visor a frequência cardíaca de quem o usa. Dessa maneira, é possível ter uma noção melhor do funcionamento do coração, já que o relógio revela a intensidade dos batimentos, se eles estão acelerados ou não. A frequência cardíaca pode dar informações valiosas sobre o corpo humano, principalmente durante exercícios.

Galton Board

A Galton Board é um dispositivo inventado por Sir Francis Galton para demonstrar o teorema do limite central, em particular, que a distribuição normal é aproximada à distribuição binomial. O tabuleiro consiste de uma placa vertical com fileiras entrelaçadas de pinos. Bolas são jogadas a partir do topo,ao bater nos pinos, elas se distribuem para a esquerda ou para a direita.

    Compra do Módulo WiFi - ESP8266

    O módulo WiFi ESP8266 é um SOC com protocolo TCP/IP integrado que consegue dar a qualquer microcontrolador acesso a sua rede WiFi. O ESP8266 é capaz tanto de hospedar uma aplicação quanto descarregar todas as funções de redes WiFi a partir de outro processador de aplicação. Cada módulo ESP8266 vai pré-programado com um comando AT setando seu firmware, significando que você pode simplesmente ligar este módulo ao seu Arduino e ele vai funcionar como qualquer outro shield WiFi funcionaria. Este módulo tem um ótimo custo x benefício e possui uma comunidade de usuários enorme, e continuamente em crescimento.

    Para automação da Galton Board utilizamos um arduino para contar as esferas que entram em cada baia. Como o arduino possui um número limitado de portas, utilizamos o mesmo ligado a dois shift register 74HCT165, o que aumenta o número de portas disponíveis para a contagem (possuimos 16 baias e o arduino UNO possui 14 portas).

Desenhos do Tabuleiro

a) Visualização do layout geral da GB: As esferas caem pela abertura entre os dois trapézios de cor cinza, mais acima. O pino de cor vermelha é o pino do primeiro choque, como orientado pela bibliografia. Os pinos vermelhos entre 1-2 e 14-15 são os pinos nos quais se dá a sequencia de choques mais a direita ou mais a esquerda, sendo o limite teórico dos choques.


b) Detalhe das baias: Detalhe das baias onde as esferas são capturadas, onde pode ser visto o encaixe das barreiras e um furo logo abaixo, através do qual os sensores de contagem das esferas são posicionados.

Board Test

Desempenho da Board

a)Teste com as esferas de diametro 10mm (maiores): Como a quantidade de esferas maiores não é grande (33 10mm esferas), o experimento não demonstrou uma curva muito acentuada. Mas é possível acompanhar, à esquerda e à direita, a diminuição do número de esferas, com uma maior concentração na região central. Além disso, o diâmetro das esferas proporciona, teoricamente, um maior número de choques, pois os espaços entre os pinos é proporcionalmente menor para a sua passagem.

b)Teste com as esferas de diametro 8mm: A quantidade um pouco maior de esferas deste diâmetro (78 8mm esferas) não melhorou a curva, que não ficou muito acentuada. Neste teste também foi possível acompanhar a diminuição do número de esferas na região mais afastada do centro.

c)Teste com as esferas de diametro 6mm:A quantidade de esferas neste experimento (186 6mm esferas) pareceu indicar que, quanto maior o número de esferas, mais aproximada a distribuição fica da curva normal. Isto é condizente com as fontes pesquisadas, pois quando o número de esferas tende ao infinito a curva tende a uma curva normal. A concentração na região central é visualmente maior do que nas regiões laterais, diminuindo gradativamente nestas últimas direções.

d)Teste com as esferas de diametro 3mm (menores):A quantidade de esferas neste experimento (~663 4mm esferas) e o diâmetro das mesmas pareceram bastante adequados para a demonstração da distribuição. O resultado foi uma figura bem semelhante a uma distribuição normal, com uma concentração maior ao centro e que, gradativamente, diminui para as laterais. Este diâmetro de esferas proporciona, teoricamente, um número menor de choques, pois o espaço entre os pinos para a passagem das esferas é proporcionalmente maior do que das demais esferas, contribuindo para o aumento da concentração na região central.

Dados do arduino Nano para o Display

Gráfico com os batimentos cardíacos na plotter serial

Adafruit_GFX.h, Adafruit_SPITFT.h e Adafruit_SSD1306.h = permitem que você trabalhe tanto com texto como com gráficos.
Oled_Address 0X3C = tipo de display
Variáveis:
x: variável para contar o tamanho da tela
lastx: posição do x na tela de gráfico
lasty: posição do y na tela de gráfico
LastTime: última vez
BMPTiming: cronometragem da BPM
BeatComplete: Batida Completa
BMP: variável que imprime o valor dos batimentos por mininuto.
UpperThreshold: Limite superior
LowerThreshold: Limite inferior

No void setup, o código é para limpar o monitor e definir o tamanho do texto;

Void Loop: O x verifica se é maior que 127, vendo se o gráfico foi além do fim da tela, se sim redefinimos de volta para o ínico da tela.
Imprimindo o Gráfico:
A variável value recebe o valor do pino de entrada 0, que é o início do batimento cardíaco. Y é o calculo da batida. O value é dividido por 16 para centralizar o gráfico no display. Oled.writeLine escreve o gráfico na tela.

Imprimindo o Batimento no Display
Quando o inicio do batimento cardíaco passa um certo limiar superior e a pulsação está completa e também se foi um limiar mais baixo estamos realmente no tempo que o coração bate por minuto, daí calculamos o batimento e exibimos na tela.

Placa NodeMCU ESP8622

Teste com sensor, display e Node

Este sensor foi desenvolvido para verificar a frequência cardíaca combinado um sensor óptico com amplificador, que envia o sinal analógico para o microcontrolador (Arduino, PIC, AVR etc), processar as informações para aplicações diversas. Se trata de um produto compacto, de excelente desempenho e fácil utilização.

Descrição Técnica:
Alimentação 3,3V ou 5V;
Ligação:
S = Sinal+
Vcc = (3,3V ou 5V)
Sinal - = GND.

Montando o circuito:
NodeMCU para display:
3V3 ------ UCC
GND ------ GND
D2 ------ SDA
D1 ------ SCL
NodeMCU para SENSOR:
A0 ------ S
GND ------ GND
3V3 ------ VCC
